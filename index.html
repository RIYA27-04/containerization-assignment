Step-by-Step Setup

1
Create project directory structure
Create the main working directory and subdirectories for backend, database, and screenshots.
mkdir containerized-webapp cd containerized-webapp mkdir backend database screenshots
2
Create all project files
Create app.py, requirements.txt, Dockerfiles, init.sql, docker-compose.yml and .gitignore.
nano backend/app.py nano backend/requirements.txt nano backend/Dockerfile nano backend/.dockerignore nano database/Dockerfile nano database/init.sql nano docker-compose.yml
3
Create IPvlan network
IPvlan shares the host MAC address — more scalable than Macvlan in virtualized/cloud environments. Must be created before running compose.
docker network create -d ipvlan \ --subnet=172.22.208.0/24 \ --gateway=172.22.208.1 \ -o ipvlan_mode=l2 \ -o parent=eth0 mynetwork
4
Verify network creation
Confirm the IPvlan network appears in the list and inspect its configuration.
docker network ls docker network inspect mynetwork
5
Build and start all containers
Docker Compose builds both images and starts containers. Backend waits for DB health check before starting.
docker compose up --build
6
Verify running containers and static IPs
Confirm both containers are up and assigned their correct static IP addresses.
docker ps docker inspect backend_api | grep IPAddress docker inspect postgres_db | grep IPAddress
7
Test API endpoints
Run health check, insert a record via POST, then fetch all records via GET.
# Health check docker exec backend_api python3 -c "import urllib.request; print(urllib.request.urlopen('http://localhost:8000/health').read().decode())" # Insert record (POST) docker exec backend_api python3 -c " import urllib.request, json data = json.dumps({'data': 'Hello World'}).encode() req = urllib.request.Request('http://localhost:8000/api/records', data=data, headers={'Content-Type': 'application/json'}) print(urllib.request.urlopen(req).read().decode())" # Fetch all records (GET) docker exec backend_api python3 -c "import urllib.request; print(urllib.request.urlopen('http://localhost:8000/api/records').read().decode())"
8
Test volume persistence
Stop containers, restart them, and verify data survives — confirming named volume works correctly.
docker compose down docker compose up -d docker volume ls # Fetch records again — data should still exist

Source Code

backend/app.py Python
from fastapi import FastAPI from pydantic import BaseModel import psycopg2, os, time app = FastAPI(title="Containerized Web App", version="1.0.0") DB_HOST = os.getenv("DB_HOST", "db") DB_PORT = os.getenv("DB_PORT", "5432") DB_NAME = os.getenv("DB_NAME", "appdb") DB_USER = os.getenv("DB_USER", "shreya") DB_PASS = os.getenv("DB_PASS", "shreya123") def get_connection(): retries = 5 while retries > 0: try: conn = psycopg2.connect(host=DB_HOST, port=DB_PORT, dbname=DB_NAME, user=DB_USER, password=DB_PASS) return conn except psycopg2.OperationalError: retries -= 1 time.sleep(2) raise Exception("Could not connect to database.") @app.on_event("startup") def startup_event(): conn = get_connection() cur = conn.cursor() cur.execute("""CREATE TABLE IF NOT EXISTS records ( id SERIAL PRIMARY KEY, data TEXT NOT NULL, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP )""") conn.commit(); cur.close(); conn.close() class RecordIn(BaseModel): data: str @app.get("/health") def health_check(): return {"status": "healthy", "service": "fastapi-backend"} @app.post("/api/records") def insert_record(record: RecordIn): conn = get_connection() cur = conn.cursor() cur.execute("INSERT INTO records (data) VALUES (%s) RETURNING id", (record.data,)) new_id = cur.fetchone()[0] conn.commit(); cur.close(); conn.close() return {"message": "Record inserted successfully", "id": new_id} @app.get("/api/records") def fetch_records(): conn = get_connection() cur = conn.cursor() cur.execute("SELECT id, data, created_at FROM records ORDER BY id") rows = cur.fetchall() cur.close(); conn.close() return [{"id": r[0], "data": r[1], "created_at": str(r[2])} for r in rows]
backend/Dockerfile Dockerfile · Multi-stage
# Stage 1: Builder — installs all dependencies FROM python:3.11-slim AS builder WORKDIR /build COPY requirements.txt . RUN pip install --upgrade pip && \ pip install --prefix=/install --no-cache-dir -r requirements.txt # Stage 2: Runtime — copies only what is needed FROM python:3.11-slim AS runtime WORKDIR /app COPY --from=builder /install /usr/local COPY app.py . # Create non-root user for security RUN addgroup --system appgroup && \ adduser --system --ingroup appgroup appuser USER appuser EXPOSE 8000 CMD ["uvicorn", "app:app", "--host", "0.0.0.0", "--port", "8000"]
docker-compose.yml YAML
version: "3.9" services: db: build: context: ./database dockerfile: Dockerfile container_name: postgres_db environment: POSTGRES_USER: shreya POSTGRES_PASSWORD: shreya123 POSTGRES_DB: appdb volumes: - pgdata:/var/lib/postgresql/data networks: mynetwork: ipv4_address: 172.22.208.10 restart: unless-stopped healthcheck: test: ["CMD-SHELL", "pg_isready -U shreya -d appdb"] interval: 10s timeout: 5s retries: 5 backend: build: context: ./backend dockerfile: Dockerfile container_name: backend_api environment: DB_HOST: 172.22.208.10 DB_PORT: 5432 DB_NAME: appdb DB_USER: shreya DB_PASS: shreya123 ports: - "8000:8000" networks: mynetwork: ipv4_address: 172.22.208.11 depends_on: db: condition: service_healthy restart: unless-stopped volumes: pgdata: name: containerized-webapp_pgdata networks: mynetwork: external: true
database/init.sql SQL
DO $$ BEGIN CREATE USER shreya WITH PASSWORD 'shreya123'; EXCEPTION WHEN duplicate_object THEN RAISE NOTICE 'User already exists, skipping.'; END $$; GRANT ALL PRIVILEGES ON DATABASE appdb TO shreya; \connect appdb; CREATE TABLE IF NOT EXISTS records ( id SERIAL PRIMARY KEY, data TEXT NOT NULL, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO shreya; GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO shreya;

API Documentation

GET
/health
Health check endpoint — used by Docker Compose healthcheck to confirm the service is running.
{"status": "healthy", "service": "fastapi-backend"}
POST
/api/records
Insert a new record into PostgreSQL. Accepts JSON body with a data field.
Request: {"data": "Hello World"} Response: {"message": "Record inserted successfully", "id": 1}
GET
/api/records
Fetch all records from the database ordered by ID ascending.
[{"id": 1, "data": "Hello World", "created_at": "2024-01-01 10:00:00"}]
GET
/docs
Auto-generated interactive Swagger UI provided by FastAPI. Available at http://localhost:8000/docs

IPvlan vs Macvlan

FeatureMacvlanIPvlan
MAC AddressUnique per containerShared with host MAC
Network LoadHigher — switch tracks many MACsLower
ScalabilityLimited by switch MAC table sizeHighly scalable
Host IsolationHost cannot reach containersHost can communicate
Best Use CaseSmall LAN / bare metal✓ Cloud / VMs — used here
